# Инструмент сверки данных (Data Reconciliation Tool)

## 📋 Описание
Jupyter Notebook для выверки данных между **Oracle** (Источник) и **Postgres** (Приемник) в рамках реализации КХД (Форма 110).  
Предназначен для обработки больших объемов данных (3–5 млн строк) без первичных ключей и индексов.

## ✨ Возможности
- **Гибкая конфигурация**: разные схемы и имена таблиц для Oracle и Postgres
- **Автоматическое определение типов полей**: числовые, строковые, даты
- **Двухэтапная сверка**:
  - Макро-уровень: агрегация COUNT(*) и SUM() по бизнес-ключам
  - Микро-уровень: детальное сравнение через `pandas.compare()`
- **Умная фильтрация**: 
  - `sample_size` — случайная выборка уникальных ключей
  - `specific_keys` — точечная проверка конкретных значений
  - `exclusions` — исключение технических полей
- **Нормализация данных**:
  - Обработка NULL через COALESCE/NVL
  - Округление чисел до 2 знаков
  - Приведение строк к верхнему регистру с обрезкой пробелов
  - Даты в формат YYYY-MM-DD
- **Визуализация**:
  - Heatmap статусов сверки (OK/MISMATCH/ONLY_IN_SOURCE/ONLY_IN_TARGET)
  - Сводная таблица с количеством ошибок

## 🚀 Быстрый старт

### 1. Установка зависимостей
```bash
pip install pandas sqlalchemy oracledb psycopg2-binary seaborn matplotlib jupyter
```

### 2. Запуск Notebook
```bash
jupyter notebook data_reconciliation_tool.ipynb
```

### 3. Настройка конфигурации
Откройте **Блок 1** в Notebook и заполните параметры:

```python
CONFIG = {
    "oracle": {
        "connection_string": "oracle+oracledb://user:pass@host:port/service_name",
        "schema": "ORA_SCHEMA",
        "table": "SOURCE_TABLE"
    },
    "postgres": {
        "connection_string": "postgresql://user:pass@host:port/dbname",
        "schema": "PG_SCHEMA",
        "table": "TARGET_TABLE"
    },
    "composite_keys": ["BANK_CODE", "REPORT_DATE"],
    "exclusions": ["ID", "LOAD_DATE", "TECH_FIELD"],  # или None для всех полей
    "sample_size": 1000,  # или None для полной проверки
    "specific_keys": None  # или {"BANK_CODE": ["001", "002"]}
}
```

### 4. Выполнение ячеек
Запустите все ячейки последовательно:
1. **Настройка** — конфигурация и подключения
2. **Сбор метаданных** — автоматическое определение типов полей
3. **Агрегация** — макро-сверка (COUNT/SUM)
4. **Визуализация** — heatmap и сводная таблица
5. **Детальный анализ** — микро-сверка проблемных записей

## 📊 Пример вывода
- **Heatmap**: визуальное отображение расхождений по датам и банкам
- **Summary Table**: статистика по типам расхождений
- **Diff Report**: детальный список различий в значениях полей

## ⚙️ Технические требования
- Python 3.8+
- Библиотеки: `pandas`, `sqlalchemy`, `seaborn`, `matplotlib`
- Драйверы БД: `oracledb`, `psycopg2-binary`
- Объем данных: оптимизировано для 3–5 млн строк
- Память: тяжелые операции только для проблемных батчей или выборки
